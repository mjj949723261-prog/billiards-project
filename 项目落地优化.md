# 《双人在线台球：产品落地与上线全景路线图》

## 一、 产品定义与核心价值
本产品是一款基于 Web 技术的轻量级、跨平台双人在线台球竞技游戏。
*   **核心痛点解决**：无需下载、即开即玩、在极端网络环境下保持高度同步与物理丝滑感。
*   **技术护城河**：基于 STOMP 的指令中转架构 + 周期性权威状态纠偏 + Delta-Time 物理模拟。

---

## 二、 研发里程碑：当前状态 (MVP Phase)
目前已完成核心玩法的闭环，并攻克了 Web 实时同步的三大技术天险：
1.  **物理真实度**：实现了非线性摩擦力模型（比例+线性）与库边切向摩擦，手感贴近现实。
2.  **视觉强一致性**：实现了 3D 旋转矩阵同步与 100ms 服务端权威广播，彻底解决了“不同设备看到球位不同”的问题。
3.  **极端防抖优化**：通过“回合感知过滤”和“EMA 插值”算法，消除了移动端常见的网络拉扯感。

---

## 三、 上线必经之路：产品加固 (Phase 1)
在正式面向外部用户推广前，需完成以下底层加固：

### 1. 身份与账户系统
*   **现状**：随机生成的 `playerId`，刷新/关浏览器即失。
*   **落地方案**：接入统一认证（JWT），支持微信/第三方登录，实现战绩与 ID 的永久绑定。

### 2. 分布式房间管理 (Redis)
*   **现状**：房间数据在内存，服务重启即丢。
*   **落地方案**：使用 Redis 存储房间对象，利用 Redis Pub/Sub 实现跨节点的 WebSocket 消息分发，支撑横向扩容。

### 3. 轻量化防作弊审计
*   **落地方案**：在不增加服务端物理计算的前提下，增加后端接口校验：
    *   **回合校验**：非当前玩家发送的击球指令一律丢弃。
    *   **力度校验**：击球 `powerRatio` 超过 1.0 的指令判定为异常。
    *   **双端互校验**：两名玩家同时上报结算状态，服务端比对关键坐标分歧。

---

## 四、 用户留存与增长：社交生态 (Phase 2)
### 1. 智能匹配系统 (Elo)
*   **功能**：不再依赖手动输入房间号。引入 Elo 算法，根据玩家胜率自动撮合旗鼓相当的对手。

### 2. 实时社交反馈
*   **功能**：预设快捷语（如“好球！”、“抱歉”）、动态 Emoji 气泡。
*   **观战模式**：允许通过分享房间链接，让更多玩家进入“观察者”席位。

### 3. 视觉表现进阶
*   **功能**：增加球杆动态光效、进球时的屏幕震动感、更精细的物理碰撞音效。

---

## 五、 商业化闭环：营收模型 (Phase 3)
### 1. 个性化商城
*   **道具**：限量款球杆皮肤、定制款台泥颜色、数字球贴图包。
*   **特效**：击球时的粒子拖尾效果。

### 2. 赛事体系
*   **功能**：开启定时锦标赛，获胜者赢取积分或徽章奖励，构建排行榜荣誉体系。

---

## 六、 运维与部署建议
1.  **静态资源 CDN**：将 `assets.js` 和渲染引擎托管至 CDN，缩短首屏加载时间。
2.  **WSS 全加密**：上线必须配置 SSL，将 WebSocket 链路升级为加密的 WSS 协议。
3.  **链路压测**：使用模拟工具（如 JMeter 或自定义脚本）进行 WebSocket 连接数压力测试，确保服务端 `ThreadPoolTaskScheduler` 的承载上限。
