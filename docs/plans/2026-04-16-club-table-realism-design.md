# 俱乐部实木桌拟真设计

## 目标

在不破坏当前房间页横屏语义适配、`2:1` 布面比例和现有物理逻辑的前提下，把对局页球桌从“程序绘制的基础桌面”提升到“俱乐部实木桌”风格。

## 设计原则

- 保持现有 `TABLE_WIDTH / TABLE_HEIGHT` 与袋口、碰撞、球路逻辑不变。
- 保持当前房间页 UI 布局、可视区计算和手机映射规则不变。
- 真实感来自“材质层”和“光影层”，而不是修改桌面几何或适配策略。
- 优先增强静态桌面观感，不引入重型运行时开销。

## 视觉方向

### 1. 实木外框

- 目标观感：偏台球俱乐部的深棕实木，不走电竞霓虹风。
- 关键特征：
  - 深棕木色主体
  - 外沿较暗、内沿较亮，形成倒角感
  - 轻微木纹，不做夸张纹理
  - 桌体下方有更扎实的接触阴影

### 2. 台呢

- 目标观感：经典俱乐部绿，不荧光、不偏蓝。
- 关键特征：
  - 中区略亮、边缘略暗
  - 轻微绒面方向感
  - 很轻的使用痕迹或亮度起伏，避免“纯色块”

### 3. 库边

- 目标观感：比台呢更深、更厚、更有实体感。
- 关键特征：
  - 与台呢区分明显
  - 靠台呢一侧有轻高光
  - 靠木框一侧更暗，形成包边感

### 4. 袋口

- 目标观感：真实桌口，而不是黑圆洞。
- 关键特征：
  - 袋口边缘有皮口/切口感
  - 袋口内部保留深洞感
  - 中袋和角袋的存在感要统一

## 技术方案

采用“现有几何 + 内置程序纹理”的混合方案：

- 保留现有几何尺寸、袋口坐标和主渲染容器。
- 在 `PixiRenderer` 的静态桌面绘制中增加材质纹理层和高光/阴影层。
- 在 `table-renderer.js` 的 Canvas fallback 中同步保留相同视觉层次，确保两条渲染路径风格接近。

## 分层结构

从外到内分 5 层：

1. 外框阴影层  
   让整桌更“压”在场景上。

2. 实木框层  
   负责桌体主材质、木色和倒角高光。

3. 库边层  
   负责厚度、暗部和边缘高光。

4. 台呢层  
   负责主色、绒面质感和轻微明暗变化。

5. 袋口层  
   负责洞口深度、口沿阴影和皮口/切角感。

## 风险控制

- 不改常量：`TABLE_WIDTH`、`TABLE_HEIGHT`、`RAIL_THICKNESS`、`POCKET_RADIUS` 不改。
- 不改适配：`game.js` 的可用区计算与手机映射逻辑不改。
- 不改交互：瞄准线、碰撞效果、白球摆放提示和球杆逻辑不改。
- 控制性能：静态纹理优先一次生成复用，不每帧重绘复杂材质。

## 验收标准

- 第一眼明显更像真实俱乐部实木桌，而不是纯色程序块。
- 球桌仍保持当前 `2:1` 舞台比例与可视大小策略。
- 横拿和竖拿手机下的同一套横屏语义画面不被破坏。
- 不出现桌口错位、库边厚度异常、袋口遮挡球体等回归。
