## 第二阶段：视觉物理分离

### 目标
引入双层坐标系统，物理层负责计算，渲染层平滑追赶，消除网络抖动。

### 修改文件：`daily-billiards-vanilla_web/src/entities/ball.js`

#### 1. 修改 constructor

```javascript
constructor(x, y, radius, color, number) {
  // 物理层（权威状态）
  this.physicsPos = new Vec2(x, y);
  this.physicsVel = new Vec2(0, 0);
  this.physicsRot = mat3RotateColMajor(0, 1, 0, -Math.PI / 2);

  // 渲染层（平滑显示）
  this.renderPos = new Vec2(x, y);
  this.renderRot = mat3RotateColMajor(0, 1, 0, -Math.PI / 2);

  // 兼容性别名（指向物理层）
  this.pos = this.physicsPos;
  this.vel = this.physicsVel;
  this.rotMat = this.physicsRot;

  this.radius = radius;
  this.color = color;
  this.number = number;
  this.pocketed = false;
  this.mass = 1.0;
}
```

#### 2. 修改 update() 方法

将所有 `this.pos` 改为 `this.physicsPos`，`this.vel` 改为 `this.physicsVel`，`this.rotMat` 改为 `this.physicsRot`：

```javascript
update(dt) {
  if (this.pocketed) return;

  // 物理层更新
  this.physicsPos.x += this.physicsVel.x * dt;
  this.physicsPos.y += this.physicsVel.y * dt;

  const speed = Math.sqrt(
    this.physicsVel.x * this.physicsVel.x + 
    this.physicsVel.y * this.physicsVel.y
  );

  if (speed > 0.01) {
    const friction = 0.98;
    this.physicsVel.x *= friction;
    this.physicsVel.y *= friction;

    const angularVel = speed / this.radius;
    const axis = new Vec3(-this.physicsVel.y, 0, this.physicsVel.x);
    const len = Math.sqrt(axis.x * axis.x + axis.z * axis.z);
    if (len > 0.0001) {
      axis.x /= len;
      axis.z /= len;
      const rotDelta = mat3RotateColMajor(axis.x, axis.y, axis.z, angularVel * dt);
      this.physicsRot = mat3Multiply(rotDelta, this.physicsRot);
    }
  } else {
    this.physicsVel.x = 0;
    this.physicsVel.y = 0;
  }
}
```

#### 3. 添加 updateRender() 方法

```javascript
updateRender(smoothFactor = 0.25) {
  if (this.pocketed) return;

  // 渲染位置平滑追赶物理位置
  const dx = this.physicsPos.x - this.renderPos.x;
  const dy = this.physicsPos.y - this.renderPos.y;
  this.renderPos.x += dx * smoothFactor;
  this.renderPos.y += dy * smoothFactor;

  // 渲染旋转平滑追赶物理旋转
  for (let i = 0; i < 9; i++) {
    this.renderRot[i] += (this.physicsRot[i] - this.renderRot[i]) * smoothFactor;
  }
}
```

#### 4. 添加 syncPhysicsToRender() 方法

```javascript
syncPhysicsToRender() {
  // 用于停球后立即同步，避免缓慢追赶
  this.renderPos.x = this.physicsPos.x;
  this.renderPos.y = this.physicsPos.y;
  for (let i = 0; i < 9; i++) {
    this.renderRot[i] = this.physicsRot[i];
  }
}
```

## 第三阶段：固定步长物理

### 目标
消除帧率依赖，确保物理模拟在不同设备上一致。

### 修改文件：`daily-billiards-vanilla_web/src/core/physics.js`

#### 1. 定义固定步长常量

```javascript
const FIXED_TIMESTEP = 1 / 120; // 120Hz 物理更新
const MAX_SUBSTEPS = 5; // 防止死亡螺旋
```

#### 2. 修改 updateGamePhysics()

```javascript
let physicsAccumulator = 0;

export function updateGamePhysics(balls, table, dt) {
  physicsAccumulator += dt;

  let steps = 0;
  while (physicsAccumulator >= FIXED_TIMESTEP && steps < MAX_SUBSTEPS) {
    // 固定步长更新
    balls.forEach(ball => ball.update(FIXED_TIMESTEP));
    handleBallCollisions(balls);
    handleTableCollisions(balls, table);
    
    physicsAccumulator -= FIXED_TIMESTEP;
    steps++;
  }

  // 如果累积时间过多，直接丢弃（防止卡顿后追帧）
  if (physicsAccumulator > FIXED_TIMESTEP * 2) {
    physicsAccumulator = 0;
  }
}
```

### 修改文件：`daily-billiards-vanilla_web/src/entities/ball.js`

## 第四阶段：多球碰撞迭代求解

### 目标
处理多球同时碰撞的链式约束，消除穿透和抖动。

### 修改文件：`daily-billiards-vanilla_web/src/core/physics.js`

#### 1. 重写 handleBallCollisions()

```javascript
const COLLISION_ITERATIONS = 4; // 迭代次数
const POSITION_CORRECTION = 0.8; // 位置修正强度

export function handleBallCollisions(balls) {
  for (let iter = 0; iter < COLLISION_ITERATIONS; iter++) {
    for (let i = 0; i < balls.length; i++) {
      if (balls[i].pocketed) continue;

      for (let j = i + 1; j < balls.length; j++) {
        if (balls[j].pocketed) continue;

        const dx = balls[j].physicsPos.x - balls[i].physicsPos.x;
        const dy = balls[j].physicsPos.y - balls[i].physicsPos.y;
        const distSq = dx * dx + dy * dy;
        const minDist = balls[i].radius + balls[j].radius;

        if (distSq < minDist * minDist && distSq > 0.0001) {
          const dist = Math.sqrt(distSq);
          const overlap = minDist - dist;

          // 归一化碰撞法向量
          const nx = dx / dist;
          const ny = dy / dist;

          // 位置修正（按质量分配）
          const totalMass = balls[i].mass + balls[j].mass;
          const correction = overlap * POSITION_CORRECTION;
          balls[i].physicsPos.x -= nx * correction * (balls[j].mass / totalMass);
          balls[i].physicsPos.y -= ny * correction * (balls[j].mass / totalMass);
          balls[j].physicsPos.x += nx * correction * (balls[i].mass / totalMass);
          balls[j].physicsPos.y += ny * correction * (balls[i].mass / totalMass);

          // 速度修正（仅第一次迭代）
          if (iter === 0) {
            const relVelX = balls[j].physicsVel.x - balls[i].physicsVel.x;
            const relVelY = balls[j].physicsVel.y - balls[i].physicsVel.y;
            const relVelDotN = relVelX * nx + relVelY * ny;

            // 只处理接近的碰撞
            if (relVelDotN < 0) {
              const restitution = 0.95; // 弹性系数
              const impulse = -(1 + restitution) * relVelDotN / totalMass;

              balls[i].physicsVel.x -= impulse * nx * balls[j].mass;
              balls[i].physicsVel.y -= impulse * ny * balls[j].mass;
              balls[j].physicsVel.x += impulse * nx * balls[i].mass;
              balls[j].physicsVel.y += impulse * ny * balls[i].mass;
            }
          }
        }
      }
    }
  }
}
```

#### 2. 改进 handleTableCollisions()

```javascript
export function handleTableCollisions(balls, table) {
  balls.forEach(ball => {
    if (ball.pocketed) return;

    const minX = table.minX + ball.radius;
    const maxX = table.maxX - ball.radius;
    const minY = table.minY + ball.radius;
    const maxY = table.maxY - ball.radius;

    // X 轴边界
    if (ball.physicsPos.x < minX) {
      ball.physicsPos.x = minX;
      ball.physicsVel.x = Math.abs(ball.physicsVel.x) * 0.8;
    } else if (ball.physicsPos.x > maxX) {
      ball.physicsPos.x = maxX;
      ball.physicsVel.x = -Math.abs(ball.physicsVel.x) * 0.8;
    }

    // Y 轴边界
    if (ball.physicsPos.y < minY) {
      ball.physicsPos.y = minY;
      ball.physicsVel.y = Math.abs(ball.physicsVel.y) * 0.8;
    } else if (ball.physicsPos.y > maxY) {
      ball.physicsPos.y = maxY;
      ball.physicsVel.y = -Math.abs(ball.physicsVel.y) * 0.8;
    }
  });
}
```

## 第五阶段：停球后软性对账

### 目标
在球停止后，非击球方采用服务器快照，但通过渲染层平滑过渡，避免跳变。

### 修改文件：`daily-billiards-vanilla_web/src/game.js`

#### 修改 applyGameStateSnapshot()

```javascript
applyGameStateSnapshot(snapshot) {
  // 过滤运动中的 live 状态
  if (snapshot.isLive) {
    return;
  }

  // 停球后对账
  snapshot.balls.forEach((ballData, i) => {
    const ball = this.balls[i];

    // 更新物理层（权威状态）
    ball.physicsPos.x = ballData.x;
    ball.physicsPos.y = ballData.y;
    ball.physicsVel.x = ballData.vx || 0;
    ball.physicsVel.y = ballData.vy || 0;
    ball.pocketed = ballData.pocketed || false;

    // 计算偏差
    const dx = ball.physicsPos.x - ball.renderPos.x;
    const dy = ball.physicsPos.y - ball.renderPos.y;
    const deviation = Math.sqrt(dx * dx + dy * dy);

    // 偏差小于阈值：平滑追赶
    // 偏差大于阈值：立即同步（防止累积误差）
    const SOFT_SYNC_THRESHOLD = ball.radius * 2;
    if (deviation > SOFT_SYNC_THRESHOLD) {
      ball.syncPhysicsToRender();
    }
    // 否则让 updateRender() 自然追赶
  });

  // 更新游戏状态
  this.currentPlayer = snapshot.currentPlayer;
  this.gamePhase = snapshot.gamePhase;
}
```

## 注意事项

1. **兼容性别名**：`this.pos`、`this.vel`、`this.rotMat` 指向物理层，确保现有代码不受影响
2. **渲染层独立**：所有绘制代码使用 `renderPos`/`renderRot`
3. **停球判定**：速度 < 0.01 时认为停球
4. **网络同步**：白球摆放仍使用 `isLive=true` 实时同步
5. **性能监控**：固定步长可能增加 CPU 负载，注意监控

## 后续优化方向

1. **输入同步**：击球参数（力度、角度）同步，双方独立模拟
2. **种子同步**：随机数种子同步，确保物理完全一致
3. **权威服务器**：服务端运行物理引擎，客户端仅预测
4. **快照插值**：非击球方采用快照插值而非本地物理

当前方案优先解决视觉问题，长期方案需根据实际效果评估。
